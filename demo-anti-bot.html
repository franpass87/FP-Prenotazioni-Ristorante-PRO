This demo shows the anti-bot protection features implemented in the restaurant booking form.

Active Protection Mechanisms

  • Honeypot Field: Hidden field that only bots fill
  • Timestamp Validation: Detects too-fast submissions
  • User Agent Analysis: Identifies bot patterns
  • Field Pattern Analysis: Detects fake data
  • Rate Limiting: Prevents spam from same IP
  • Conditional reCAPTCHA v3: Human verification when suspicious

How It Works

1. Honeypot Detection

// Hidden field in form <div style="position: absolute; left: -9999px; visibility: hidden;"> <input type="text" name="rbf_website" tabindex="-1"> </div> // Server-side check if (!empty($_POST['rbf_website'])) { // Bot detected - block submission }

2. Timestamp Validation

// Check submission timing $submission_time = time() - $form_timestamp; if ($submission_time < 5) { // Too fast - likely bot $suspicion_score += 80; }

3. Conditional reCAPTCHA

// Only trigger reCAPTCHA for suspicious submissions if ($bot_score >= 70 && $recaptcha_configured) { $result = verify_recaptcha($_POST['g-recaptcha-response']); if (!$result['success']) { // Block submission } }
